4. Create `web/src/lib/moderation/moderateText.ts` — imports the shared OpenAI singleton instead of creating its own instance:
   ```typescript
   import { openai } from "../openai";

   export interface ModerationResult {
     flagged: boolean;
     categories: string[];
   }

   export async function moderateText(text: string): Promise<ModerationResult> {
     const response = await openai.moderations.create({
       model: "omni-moderation-latest",
       input: text,
     });
     const result = response.results[0];
     const flaggedCategories = Object.entries(result.categories)
       .filter(([, flagged]) => flagged)
       .map(([category]) => category);
     return { flagged: result.flagged, categories: flaggedCategories };
   }
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client form → Server Action | User-supplied PLZ, email, issue text crosses into server-side processing |
| Server → OpenAI Moderation API | User text sent to external API for content classification |
| Server → data files | Static JSON read from filesystem — trusted but must validate structure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | PLZ input | mitigate | Validate PLZ with Zod regex `/^\d{5}$/` + `.refine(v => parseInt(v) >= 1001)` at server boundary in wizardSchemas.ts — reject non-numeric, wrong length, and out-of-range values |
| T-02-02 | Tampering | Email input | mitigate | Validate email with Zod `.email()` server-side — client validation is a UX convenience, server validation is the security gate |
| T-02-03 | Information Disclosure | OpenAI API key | mitigate | Key accessed only via `process.env.OPENAI_API_KEY` in server-side singleton (web/src/lib/openai.ts) — never imported in client components |
| T-02-04 | Denial of Service | PLZ lookup | accept | Static JSON file read with module-scope caching; file is ~1MB max; no amplification vector |
</threat_model>
