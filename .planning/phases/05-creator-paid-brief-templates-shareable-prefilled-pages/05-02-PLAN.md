<tasks>

<task type="auto">
  <name>Task 1: Build the creator setup form and draft creation action</name>
  <action>
    Add the public entry route and form for campaign creation.

    The form must collect:

    - creator email
    - campaign title
    - prefilled issue text
    - optional creator or NGO name
    - optional short description
    - optional external link
    - chosen slug

    The server action must:

    - validate and normalize all fields
    - moderate `issue_text` and `description`
    - reserve the slug by creating a draft campaign row
    - reject blocked content with a clear German error
    - create a Stripe Checkout session and return its redirect target
  </action>
  <verify>
    <automated>cd web && npx tsc --noEmit && rg -n "creator email|slug|moderateText|checkout" src</automated>
  </verify>
</task>

<task type="auto">
  <name>Task 2: Wire Stripe webhook to paid campaign state</name>
  <action>
    Implement webhook handling that:

    - verifies Stripe signatures
    - matches the paid checkout session to a draft campaign
    - records payment identifiers on the campaign
    - transitions the campaign to `awaiting_email_verification`
    - triggers a creator verification email

    The webhook must be idempotent. Replayed Stripe events must not send duplicate state changes or corrupt the record.
  </action>
  <verify>
    <automated>cd web && rg -n "checkout.session.completed|awaiting_email_verification|idempotent" src/app/api/stripe src/lib/campaigns src/lib/email</automated>
  </verify>
</task>

<task type="auto">
  <name>Task 3: Add verification and creator email delivery</name>
  <action>
    Implement:

    - a verification route or action that consumes a `verify_email` token
    - activation logic that only makes the campaign `active` when payment and verification are both complete and the latest public text still passes moderation
    - a management email sent after activation with the reusable management entry link

    The email copy should prioritize:

    - campaign URL
    - activation status
    - edit/pause instructions
    - no-login explanation
  </action>
  <verify>
    <automated>cd web && npx tsc --noEmit && rg -n "verify_email|active|management" src/lib/email src/lib/actions src/app</automated>
  </verify>
</task>
