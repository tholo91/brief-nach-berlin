Purpose: D12 (newly in scope per the Discuss-Phase update on 2026-05-09) requires that the letter prompt adapts. Without this plan, PLAN 04's level routing produces the right recipient but the wrong letter content.

Output: One refactored generator function + one extended generate-letter API handoff + extended test coverage.

Anrede mapping (review W10 — no slash-form unless gender unknown; revised 2026-05-22 for Kommune):
```
gender + level → Anrede:
- gender=m, level=Bund/Land           → "Sehr geehrter Herr Abgeordneter [Nachname]"
- gender=f, level=Bund/Land           → "Sehr geehrte Frau Abgeordnete [Nachname]"
- gender=d, level=Bund/Land           → "Sehr geehrte:r [Nachname]"
- level=Kommune (alle Gemeinden!)     → "Sehr geehrte Damen und Herren der Stadtverwaltung," (Thomas-Decision 2026-05-22, see D13)
- level=Kommune, recipientKind=bezirksamt → "Sehr geehrte Damen und Herren des Bezirksamts,"
- gender unknown (Bund/Land)          → last-resort slash: "Sehr geehrte/r [Anrede] [Nachname]"
```

**Kommune is intentionally generic (D13 Thomas-Decision 2026-05-22):**
- Kein Bürgermeister-Name, kein Gender-Resolve, kein `formal_title` für die Anrede selbst.
- Adresse-Block enthält den `formal_title` als Empfänger-Label ("Stadtverwaltung Köln, Stadt").
- 100% Coverage über alle ~11.000 Gemeinden, kein Wikidata-Lookup nötig.

**Stadtstaaten special handling (review W10):**
- `bundeslandKey ∈ {HH, HB, BE}` + `level=Land` → resolved gender + formal_title (Senator, Senatorin, Erster Bürgermeister, Bürgermeisterin).
- `bundeslandKey ∈ {HH, HB}` + `level=Kommune` → routeToLevel/lookupPLZWithLevel re-routet auf Land (Einheitsgemeinde, Art. 28 Abs. 1 S. 2 GG). Wird gar nicht erst als Kommune-Brief erreicht.
- `bundeslandKey = BE` + `level=Kommune` → Bezirksamt-Variante: Anrede `"Sehr geehrte Damen und Herren des Bezirksamts,"`, formal_title `"Bezirksamt Berlin, [Bezirksname]"`.

Framing mapping, **Strategie + Nachdruck + Formalität + Argumentation pro Ebene** (revised 2026-05-22 per Thomas-Decision). **WICHTIG (LOCK-1): Bund ist NICHT enthalten. Der Bund-Branch gibt heute's `SYSTEM_PROMPT_TEMPLATE` byte-identisch zurück. Nur Land + Kommune bekommen neue Strategy-Blocks:**

**Bund (Bundestagsabgeordnete):** UNVERÄNDERT. Es wird KEIN neuer Bund-Strategy-Block eingeführt (LOCK-1). Der Bund-Branch von `buildSystemPrompt` liefert das heutige `SYSTEM_PROMPT_TEMPLATE` 1:1 (nur `__TODAY__` ersetzt). Verifiziert durch den Bund-Snapshot-Test in Task 1. Heute's Prompt enthält bereits die korrekten Bund-Regeln (strukturelle Argumentation, Fraktions-/Ausschuss-Hebel, kein "Art. X GG"), daher kein Eingriff nötig.

**Land (Landtagsabgeordnete):**
- **Strategie:** Landesspezifische Verantwortlichkeit, konkrete Landesinstrumente (Landesschulgesetz, Landespolizeigesetz, Krankenhausplan). Argumentation in Landes-Logik, nicht Bundes-Logik.
- **Formalität:** Mittlere formelle Distanz. Empfänger ist näher an der Wähler:in-Schaft als ein MdB (kleinerer Wahlkreis). Tonfall darf etwas konkreter, regionaler sein.
- **Nachdruck-Hebel:** Bezug zum konkreten Wahlkreis / Stadtteil / Ort, Verweis auf Landtags-Ausschuss-Befassung, Bürger:in-Status im Wahlkreis. Wenn passend: aktuelle Landespolitische Debatte aufgreifen.
- **Argumentations-Muster:** "[Bundesland] hat hier eigene Kompetenz und Gestaltungsspielraum. Ich erwarte, dass…" ODER "Im Bundesvergleich liegt [Bundesland] zurück / vorn, das sollte adressiert werden".
- **Verboten:** Bundesgesetzgebungs-Forderungen (das wäre Bund-Sache). Keine GG-Artikelnummern.

**Kommune (Stadtverwaltung / Bezirksamt):**
- **Strategie:** Lokale Direktheit, konkrete Verwaltungs-Einheit (Bauamt, Ordnungsamt, Verkehrsbehörde), kurzer Weg zur Lösung. Der Brief richtet sich an die Verwaltung als Ganzes, nicht an eine Einzelperson.
- **Formalität:** Geringere formelle Distanz als Bund/Land, aber weiterhin Sie-Form. Tonfall konkret, sachlich, alltagsorientiert. Empfänger ist kein Politiker mit Wahlkampf-Logik, sondern Verwaltung, also weniger ideologisch, mehr lösungs-orientiert argumentieren.
- **Nachdruck-Hebel:** Konkrete Orts-/Straßen-Angabe, Verweis auf Pflichtaufgaben der Gemeinde (Bauleitplanung, Straßenunterhaltung, ÖPNV-Zuständigkeit), Bürger:in-Status in der Gemeinde. Bei Untätigkeit: Hinweis auf öffentliche Sichtbarkeit (Lokalpresse, Stadtteil-Versammlung) als legitimen demokratischen Druck.
- **Argumentations-Muster:** "Als Bürger:in in [Ortsname] erwarte ich, dass die Stadtverwaltung [konkrete Forderung]. Diese Aufgabe liegt in der kommunalen Selbstverwaltung."
- **Verboten:** Politik-Floskel-Sprache (Mehrwert, Maßnahmenpaket, Synergien). Briefe an Verwaltung müssen NICHT politisch klingen, sie müssen handlungs-orientiert klingen.

**Stellvertretend-Mode (mode='secondary-stellvertretend', Thomas-Decision 2026-05-21):**
- Bund + secondary: explizite Anerkennung der Kompetenz-Asymmetrie. Eröffnung: *"Sehr geehrte Frau Abgeordnete Müller, ich weiß, dass Sie als Bundestagsabgeordnete für [Bildungspolitik] nicht direkt zuständig sind, diese liegt bei den Ländern. Trotzdem wende ich mich an Sie, weil [User-Begründung aus Anliegen]."*
- Land + secondary: analoge explizite Anerkennung.
- Kommune + secondary: in v1 nicht erreicht, Stadtverwaltung ist immer primary für Kommune-Anliegen.

[Falls Override: 'Du hast die Ebene manuell auf [Level] gewechselt.']"*~~ **ENTFERNT per LOCK-4c - nicht implementieren.** Es gibt KEINEN sichtbaren Routing-Footer in der Email (siehe Task 3, must_haves).
