}

  bundeslandName: string;
}): RathausRecipient;

The `RathausRecipient` carries ONLY the fields letter-generation + email actually need. It does NOT extend `Politician`, so there is no fake `id` to spoof. Field values:

| RathausRecipient field | Stadtverwaltung value | Notes |
|---|---|---|
| `kind` | `"rathaus"` | Discriminator (LOCK-5) |
| `level` | `"Kommune"` | Canonical (review B3) |
| `recipientKind` | `"stadtverwaltung"` or `"bezirksamt"` | BE uses bezirksamt |
| `gemeindeName` | `ortsname` | "Köln, Stadt" |
| `plz` | `plz` | The looked-up PLZ |
| `label` | `"Stadtverwaltung [Ortsname]"` | Anrede label + email recipient line |
| `postalAddress` | `"Stadtverwaltung [Ortsname], [PLZ] [Ortsname]"` | Generic post-tauglich (comma separator, mirrors politicians-cache) |

**Generation handoff:** at the `generateLetter` call site, the chosen `RathausRecipient` is passed with `level: "Kommune"` + its `label` + `postalAddress`. PLAN 06's Kommune branch uses the generic Anrede "Sehr geehrte Damen und Herren der Stadtverwaltung," so the generator needs no personal name and no `id`. Security is already settled in `selectPoliticianAction` before generation runs.

**Berlin special case:**
- For `bundeslandKey === "BE"`: `recipientKind = "bezirksamt"`, `lastName = "Bezirksamt"`, `formal_title = "Bezirksamt [Ortsname]"`. OpenPLZ ortsname for Berlin contains the Bezirk (e.g. "Berlin, Friedrichshain-Kreuzberg"), so this works.

**Hamburg / Bremen re-route:**
- For `bundeslandKey ∈ {"HH", "HB"}`: do NOT build a Kommune recipient. The Routing layer (`submitWizard.ts` per PLAN 04 + 06) re-routes Kommune → Land per D12. PLAN 07 just gracefully returns `null` for these.
