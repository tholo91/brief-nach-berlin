# Phase 260522-k6t Plan 01: Reviews-Seite /stimmen Summary

**One-liner:** Trust-funnel page at /stimmen with CSS marquee, anon+service-role RLS split, and dual feedback-CTA routing heyspeak vs mail-link star rating.

## What was built

Three tasks executed sequentially, each committed atomically.

### Task 1: Data helpers (028d275)
- `types.ts`: PublicReview, RatingDistribution, ReviewStats types
- `getPublicReviews`: Uses anon Supabase client -- RLS-bounded to consent=true rows, column-restricted to display-safe fields. Filters null and empty bodies in JS as a second pass.
- `getReviewStats`: Uses service role client, fetches only the `rating` column, computes avg/distribution/count in JS, returns aggregates only -- never row data.

### Task 2: Components (06c0e51)
- `ReviewCard`: Server Component, inline SVG star row (5 stars, filled/empty by rating), body with `line-clamp-4`, date in de-DE locale (e.g. "Mai 2026"), "Anonym" fallback for null display_name.
- `ReviewMarquee`: Client Component. CSS `@keyframes marquee` at 60s linear infinite, pause-on-hover via animation-play-state, `prefers-reduced-motion: reduce` switches to static overflow-x: auto. Doubles the list for seamless loop.
- `RatingStat`: Server Component, shows avg to 1 decimal + total count, optional distribution bars, "Noch zu wenig Bewertungen" fallback when totalCount === 0.

### Task 3: /stimmen page (311ff79)
- 8 sections in order: Hero, StatStrip, Feedback-CTA, ReviewMarquee, Curated Highlights, Wo wir herkommen, FAQ, Final-CTA.
- `robots: { index: false, follow: false }` -- page is live but not crawlable until Thomas approves sitemap entry.
- `revalidate = 3600` -- ISR cache of 1h.
- Three JSON-LD blocks: Article, FAQPage, AggregateRating (only when totalCount >= 5).
- Empty-state handling: if reviews.length === 0, Marquee + Highlights sections are fully hidden. RatingStat shows its own fallback text.
