# /stimmen Dynamic Count + Consolidate End Blocks

Two changes to `web/src/app/(site)/stimmen/page.tsx`. Two atomic commits.

## Task 1 — Dynamic letter count from counter

**Why:** Hardcoded "350+" goes stale within days. Footer already uses `getLetterCount()` from `@/lib/counter`. Use the same source on /stimmen so the page tells the true current number.

**Changes:**
- Add import: `import { getLetterCount } from "@/lib/counter";`
- Remove `export const revalidate = 3600;` — `getLetterCount` internally calls `noStore()`, which opts the page into dynamic rendering. Keeping `revalidate` would be misleading dead code. The page now renders per-request (low traffic, ~5k pageviews/month, well within Supabase free tier).
- Extend the `Promise.all` to fetch the counter alongside reviews + stats:
  ```tsx
  const [reviews, stats, letterCount] = await Promise.all([
    getPublicReviews(30),
    getReviewStats(),
    getLetterCount(),
  ]);
  ```
- Derive a display string with a sane fallback so the page never reads "0 Briefe" if the counter ever fails: `const displayCount = letterCount > 0 ? letterCount.toString() : "350";`
- Replace 3 hardcoded mentions:
  - Standfirst (~L274): `dreihundertfünfzig Briefe` → `{displayCount} Briefe`
  - Figure caption (~L283): `Über 350 Briefe` → `${displayCount} Briefe` (drop "Über" — exact count makes "Über" awkward)
  - Section-5 paragraph (~L286): `über 350 Briefe` → `${displayCount} Briefe` (same — drop "über")

**Commit message:** `feat(stimmen): use dynamic letter count from Supabase counter`

**Verify:** `grep -nE "(350|dreihundertfünfzig)" web/src/app/(site)/stimmen/page.tsx` shows only the fallback constant; `grep -n "getLetterCount" web/src/app/(site)/stimmen/page.tsx` shows the import and the await; `cd web && npx tsc --noEmit` exits 0.
