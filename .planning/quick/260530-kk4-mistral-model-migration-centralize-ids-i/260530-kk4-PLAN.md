# Quick Task 260530-kk4: Mistral Model Migration

## Goal

Migrate brief-nach-berlin off the soon-to-be-deprecated Mistral models, centralize model IDs in one place for future migrations, and switch letter generation to the cheaper `mistral-small-latest` v4 with `reasoning_effort: "high"`.

## Decisions (from clarifying questions in approved plan)

1. **Letter model:** Switch to `mistral-small-latest` with `reasoning_effort: "high"` (~10× cheaper than `mistral-medium-3-5`). Includes A/B validation gate before final commit.
2. **Pinning:** Use named aliases (`-latest`) in the registry. Source of truth lives in `web/src/lib/mistral.ts` so DSGVO copy can be reconciled in one spot.
3. **Pages:** Update DSGVO + KI-transparenz copy in the same pass — current copy already misstates the model.
