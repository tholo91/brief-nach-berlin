# Quick Task 260625-9at: E-Mail-Adresse aus Hardcoding befreien

## Goal

`thomas_lorenz@posteo.de` ist in 9 Stellen im Code hardkodiert. Thomas hat `THOMAS_MAIL`
als Vercel-Env-Var hinterlegt. Die Adresse soll nicht im Open-Source-Repo sichtbar sein.

`datenschutz@brief-nach-berlin.de` ist die Datenschutz-Adresse des Projekts — keine
persönliche Adresse, bleibt unberührt.

## Tasks

### Task 1: contact.ts — Single Source of Truth

**File:** `web/src/lib/contact.ts`

Ändern:
```ts
export const CONTACT = {
  email: process.env.THOMAS_MAIL ?? "thomas_lorenz@posteo.de",
} as const;
```

Fallback bleibt für lokale Entwicklung ohne .env.local-Eintrag.

---

### Task 2: config.ts — FOUNDER_EMAIL delegiert an contact.ts

**File:** `web/src/lib/config.ts`

1. `import { CONTACT } from "@/lib/contact";` am Anfang ergänzen
2. Zeile 61 ändern: `export const FOUNDER_EMAIL = CONTACT.email;`

Kein `as const` nötig — der Wert kommt jetzt aus contact.ts.

### Task 3: sendFollowupEmail.ts — replyTo via FOUNDER_EMAIL

2. Zeile 49: `replyTo: { email: FOUNDER_EMAIL }` statt hardkodierter String
