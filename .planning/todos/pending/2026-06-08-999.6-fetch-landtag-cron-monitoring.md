## Problem

Phase 999.6 D19/D20 verlangt einen wöchentlichen Cron, der `npm run fetch:landtag` ausführt und bei Cache-Änderung einen PR öffnet. Ohne aktive Überwachung kann der Workflow 3+ Wochen still ausfallen → die App schreibt Briefe an Ex-MdLs. Silent failure ist hier besonders gefährlich, weil der User es nie erfährt: der Brief geht raus, die Post stellt eventuell zu, ein Praktikant öffnet ihn und wirft ihn weg.

## Solution

Zwei Layer:

1. **Workflow-Failure-Alert (GitHub):**
   - `.github/workflows/fetch-landtag.yml` bekommt einen `on: failure` Step, der ein GitHub Issue automatisch öffnet mit Label `urgent`, Titel `[fetch-landtag] cron failed YYYY-MM-DD`, Body mit Workflow-Log-Link.
   - Issues mit Label `urgent` triggern Email-Notification an thomas-lorenz@posteo.de (über GitHub-Notification-Settings).

2. **Build-time Staleness-Check (App):**
   - In `next.config` oder einem Pre-Build-Script: lies `politicians-cache.json` → wenn `lastUpdated` älter als 14 Tage, breche den Build mit klarer Fehlermeldung ab: `politicians-cache.json ist 16 Tage alt — bitte fetch:landtag manuell laufen lassen.`
   - Verhindert, dass ein veralteter Cache versehentlich live geht, falls der Cron schweigend gestorben ist.

## Trigger

Direkt nach 999.6-Merge, bevor der erste reale Landtag-Brief rausgeht. Sicherheitskritisch, nicht optional.
